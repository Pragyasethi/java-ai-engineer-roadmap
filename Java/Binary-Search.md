#Binary Search

Binary Search repeatedly divides a sorted array into half until the element is found.

Imagine finding a word in a dictionary.

You don’t start from page 1.

You open the middle page and decide whether to go left or right.

Why do we need it?

To search much faster in sorted data.

Important Points

* Array must be sorted
* Divide into half
* Time Complexity: O(log n)

Real-world Examples

* Dictionary
* Phone contacts
* Library catalog

Memory Trick

Binary = Divide into Half
